# Chat Message Edit Feature — Learning Guide

## Goal
Show an edit button on hover over your own messages, and implement inline editing. You'll write the code, I'll explain the concepts.

---

## Part 1: How Hover Reveal Works (CSS Concept)

The key technique is **Tailwind's `group` + `group-hover`** system:

```
Parent has class "group"
  └── Child uses "opacity-0 group-hover:opacity-100"
        → Child is invisible until you hover anywhere on the parent
```

**Why this works:** Tailwind's `group` marks a parent. Any child with `group-hover:*` responds to hover on that parent, not just the child itself. This is how you get "hover over the message area, see a button" behavior without JavaScript.

### Where to apply it

In `resources/js/components/ChatMessages.vue`, each message currently looks like:

```html
<div class="flex flex-col" :class="msg.sent ? 'items-end' : 'items-start'">
    <div class="max-w-[70%] px-3 py-2 text-xs" :class="...">
        {{ msg.content }}
    </div>
    <span class="mt-1 text-[10px] text-muted-foreground">{{ msg.time }}</span>
</div>
```

You need to:
1. Add `class="group"` to the outer `<div>` (the one with `flex flex-col`)
2. Wrap the message content in another div with `relative` (so the edit button can be positioned absolutely)
3. Add the edit button with `opacity-0 group-hover:opacity-100 transition-opacity`
4. Only show it when `v-if="msg.sent"` (your own messages)

### The edit button
Look at how `IconDelete.vue` works — it imports from `@hugeicons/vue` and `@hugeicons/core-free-icons`. You'll make an `IconEdit.vue` the same way but with a pencil icon. Browse the hugeicons library for something like `Edit02Icon` or `PencilIcon`.

## Part 2: Inline Editing (The UX Pattern)

### Concept: Reactive state toggling

The idea is simple:
- `editingId` — a ref that holds `null` or the ID of the message being edited
- When `editingId === msg.id` → show a `<textarea>` instead of the message text
- When `editingId !== msg.id` → show normal message text

```
<template v-if="editingId === msg.id">
    <!-- textarea here -->
</template>
<template v-else>
    <!-- normal message here -->
</template>
```

This is Vue's conditional rendering. The `v-if` / `v-else` pair means only one is ever in the DOM.

### The textarea behavior

You'll need:
- `editContent` ref — holds the current textarea value
- `startEditing(msg)` — sets `editingId = msg.id` and `editContent = msg.content`
- `saveEdit(msg)` — sends a PUT request, updates local state
- `cancelEdit()` — resets `editingId = null`

For keyboard handling on the textarea:
- `@keydown.enter.prevent` — calls saveEdit (prevent adds a newline)
- `@keydown.escape` — calls cancelEdit

### Focus management

After clicking edit, the textarea needs to be focused. Use a template ref:
```vue
<textarea ref="editInput" ... />
```
Then in `startEditing()`, use `nextTick(() => editInput.value?.focus())` to focus after Vue renders.

## Part 3: Backend Changes

### Route
Currently only `POST /chats/{chat}/messages` exists (for creating). You need a `PUT /messages/{message}` for updating. Add it in `routes/web.php` inside the auth middleware group.

### Controller
`MessageController::update()` is already scaffolded but empty. You need to:
1. Validate the request (create an `UpdateMessageRequest` form request, or inline validation)
2. Check authorization: `$message->user_id === Auth::id()` — only the sender can edit
3. Update: `$message->update(['content' => $request->content])`
4. Return back to the chat page

### What to return
Since this is Inertia, you can either:
- **Option A:** `return to_route('chats.show', $message->chat)` — full page reload (simplest)
- **Option B:** Return JSON and update local state with `router.reload({ only: ['chat'] })` — no reload (smoother)

Option A is fine to start. You can optimize later.

### "Edited" indicator
- Add an `edited_at` column to the messages table via a migration
- Set it in the controller: `$message->update(['content' => ..., 'edited_at' => now()])`
- Add `edited_at` to the model's `$casts`
- Add `'edited' => $this->edited_at !== null` to `MessageResource`
- In the template, show a small "(edited)" text when `msg.edited` is true

## Step-by-Step Order

1. **Migration** — `php artisan make:migration add_edited_at_to_messages_table` then run it
2. **Route** — add `Route::put('/messages/{message}', ...)`
3. **Form Request** — `php artisan make:request UpdateMessageRequest`
4. **Controller** — implement `update()` with validation + authorization
5. **MessageResource** — add `edited` field
6. **Message model** — add `edited_at` to `$casts`
7. **IconEdit.vue** — copy IconDelete.vue pattern, swap icon
8. **ChatMessages.vue** — add hover reveal + inline editing logic
